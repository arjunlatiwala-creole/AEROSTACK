## 3. HIRING PIPELINE STAGES (State Machine)

```
 ╔══════════════════════════════════════════════════════════════════════╗
 ║                  CANDIDATE STAGE STATE MACHINE                      ║
 ╠══════════════════════════════════════════════════════════════════════╣
 ║                                                                      ║
 ║                      ┌──────────────┐                               ║
 ║                      │  SUBMISSION  │ (10% probability)             ║
 ║                      │              │ Form submitted, resume in S3  ║
 ║                      └──────┬───────┘                               ║
 ║                             │                                        ║
 ║                             v                                        ║
 ║                      ┌──────────────┐                               ║
 ║                      │ FIRST_TOUCH  │ Someone from Enterprise calls       ║
 ║                      │              │ Disambiguate role/purpose     ║
 ║                      └──────┬───────┘                               ║
 ║                             │                                        ║
 ║                             v                                        ║
 ║                      ┌──────────────┐                               ║
 ║                      │   NDA_SENT   │ NDA sent for signature        ║
 ║                      │              │ Must be signed before         ║
 ║                      └──────┬───────┘ interviews begin              ║
 ║                             │                                        ║
 ║                             v                                        ║
 ║                      ┌──────────────┐                               ║
 ║                      │  NDA_SIGNED  │ Candidate signed NDA          ║
 ║                      │              │ Ready for interviews          ║
 ║                      └──────┬───────┘                               ║
 ║                             │                                        ║
 ║                     ┌───────┴───────┐                               ║
 ║                     v               v                                ║
 ║              ┌─────────────┐ ┌─────────────┐                       ║
 ║              │ CULTURE_FIT │ │  ROLE_FIT   │  Can happen in        ║
 ║              │             │ │             │  parallel. Both must   ║
 ║              │ 80% culture │ │ 80% role    │  complete before       ║
 ║              │ 20% role    │ │ 20% culture │  decision gate.        ║
 ║              └──────┬──────┘ └──────┬──────┘                       ║
 ║                     │               │                                ║
 ║                     └───────┬───────┘                               ║
 ║                             │                                        ║
 ║                             v                                        ║
 ║                      ┌──────────────┐                               ║
 ║                      │   DECISION   │ Both interviews done          ║
 ║                      │              │ Team decides next step        ║
 ║                      └──┬───┬───┬───┘                               ║
 ║                         │   │   │                                    ║
 ║            ┌────────────┘   │   └────────────┐                      ║
 ║            v                v                v                       ║
 ║     ┌────────────┐  ┌────────────┐  ┌──────────────┐               ║
 ║     │  PROPOSAL  │  │  RECYCLED  │  │  ELIMINATED  │               ║
 ║     │            │  │            │  │              │               ║
 ║     │ Prepare &  │  │ Re-engage  │  │ Not a fit.   │               ║
 ║     │ send job   │  │ in 1-12    │  │ Send nice    │               ║
 ║     │ proposal   │  │ months.    │  │ rejection.   │               ║
 ║     └─────┬──────┘  │ Set date.  │  └──────────────┘               ║
 ║           │         └────────────┘                                   ║
 ║           │                                                          ║
 ║           │         ┌────────────┐                                   ║
 ║           │         │REFERRED_OUT│ Send to someone else             ║
 ║           │         └────────────┘                                   ║
 ║           v                                                          ║
 ║    ┌──────────────┐                                                 ║
 ║    │ NEGOTIATION  │ Negotiate terms, commitment call                ║
 ║    │              │ "Is this what you want?"                        ║
 ║    └──────┬───────┘                                                 ║
 ║           │                                                          ║
 ║           v                                                          ║
 ║    ┌──────────────┐                                                 ║
 ║    │  OFFER_SENT  │ Formal offer via Deel                           ║
 ║    │              │ Candidate signs in Deel                         ║
 ║    └──────┬───────┘                                                 ║
 ║           │                                                          ║
 ║           v                                                          ║
 ║    ┌──────────────┐                                                 ║
 ║    │OFFER_ACCEPTED│ Candidate signed                                ║
 ║    │              │ Trigger: create Google Workspace user           ║
 ║    │              │ Trigger: assign correct OU                      ║
 ║    │              │ Trigger: AWS Identity Center policies           ║
 ║    └──────┬───────┘                                                 ║
 ║           │                                                          ║
 ║           v                                                          ║
 ║    ┌──────────────┐                                                 ║
 ║    │  ONBOARDING  │ Moodle content assigned                        ║
 ║    │              │ 1:1 with manager scheduled                     ║
 ║    │              │ People Ops dashboard created                   ║
 ║    │              │ Early work requirements set                    ║
 ║    │              │ Certification goal requested                   ║
 ║    └──────┬───────┘                                                 ║
 ║           │                                                          ║
 ║           v                                                          ║
 ║    ┌──────────────┐                                                 ║
 ║    │    ACTIVE    │ Employee is fully onboarded                     ║
 ║    │              │ Visible in People Ops dashboard                ║
 ║    └──────────────┘                                                 ║
 ║                                                                      ║
 ╚══════════════════════════════════════════════════════════════════════╝


 ALLOWED TRANSITIONS:
 ─────────────────────
 SUBMISSION     → FIRST_TOUCH
 FIRST_TOUCH    → NDA_SENT
 NDA_SENT       → NDA_SIGNED
 NDA_SIGNED     → CULTURE_FIT | ROLE_FIT
 CULTURE_FIT    → ROLE_FIT | DECISION (if role_fit already done)
 ROLE_FIT       → CULTURE_FIT | DECISION (if culture_fit already done)
 DECISION       → PROPOSAL | RECYCLED | ELIMINATED | REFERRED_OUT
 PROPOSAL       → NEGOTIATION
 NEGOTIATION    → OFFER_SENT | RECYCLED | ELIMINATED
 OFFER_SENT     → OFFER_ACCEPTED | ELIMINATED
 OFFER_ACCEPTED → ONBOARDING
 ONBOARDING     → ACTIVE

 ANY STAGE      → ELIMINATED (candidate can drop out anytime)
 ANY STAGE      → RECYCLED (can be shelved anytime)
```

## 6. EVENT-DRIVEN AUTOMATION

```
 ═══════════════════════════════════════════════════════════════════
 EVENTBRIDGE RULES & AUTOMATIONS
 ═══════════════════════════════════════════════════════════════════

 ┌─────────────────────────────────────────────────────────────────┐
 │                                                                 │
 │  EVENT: candidate.created                                       │
 │  ─────────────────────────                                      │
 │  Fired when: new application submitted (public or internal)    │
 │                                                                 │
 │  Rule 1 → Lambda: notify-hiring-owner                          │
 │           Sends Slack message to posting's hiring owner:        │
 │           "New applicant: Jane D. for Sr Platform Engineer"     │
 │           Includes link to candidate detail in Aerostack             │
 │                                                                 │
 │  Rule 2 → Lambda: send-candidate-ack (future)                  │
 │           Sends email to candidate:                             │
 │           "Thanks for applying, we'll be in touch"              │
 │                                                                 │
 │                                                                 │
 │  EVENT: candidate.stage_changed                                 │
 │  ──────────────────────────────                                 │
 │  Fired when: stage is updated via hiring board or detail page  │
 │  Payload: { candidateId, previousStage, newStage, changedBy }  │
 │                                                                 │
 │  Rule 1 → Lambda: notify-stage-change                          │
